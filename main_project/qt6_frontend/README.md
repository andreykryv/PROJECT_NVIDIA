# SortBench Qt6 Frontend

Приложение для сравнения производительности алгоритмов сортировки на CPU и GPU, переписанное на Qt 6.

## Структура проекта

```
qt6_frontend/
├── CMakeLists.txt          # Конфигурация CMake для Qt 6
└── src/
    ├── main.cpp            # Точка входа
    ├── mainwindow.h        # Заголовок главного окна
    ├── mainwindow.cpp      # Реализация главного окна
    ├── sortingvisualizer.h # Заголовок виджета визуализации
    ├── sortingvisualizer.cpp
    ├── benchmarksimulator.h # Заголовок симулятора бенчмарков
    └── benchmarksimulator.cpp
```

## Требования

- Qt 6.2 или выше
- CMake 3.20 или выше
- Компилятор с поддержкой C++17

## Сборка

```bash
cd qt6_frontend
mkdir build && cd build
cmake ..
make -j$(nproc)
```

## Запуск

```bash
./SortBenchQt6
```

## Функционал

### Вкладка "Бенчмарки"
- Выбор алгоритмов CPU (QuickSort, MergeSort, HeapSort, TimSort, etc.)
- Выбор алгоритмов GPU (Bitonic, Radix, Odd-Even, etc.)
- Настройка размера массива (100 - 10,000,000)
- Выбор распределения данных (Uniform, Sorted, Reverse, Nearly Sorted)
- Выбор типа данных (double, float, int)
- Количество запусков для статистики
- Графики сравнения производительности
- Таблица результатов с метриками
- Экспорт в CSV и JSON

### Вкладка "Визуализация"
- Анимированная визуализация алгоритмов сортировки
- Поддержка 18+ алгоритмов
- Регулировка скорости анимации
- Настройка размера массива (10-100 элементов)
- Кнопки Старт/Пауза/Новый массив
- Цветовая индикация:
  - Синий - обычные элементы
  - Красный - сравниваемые элементы
  - Оранжевый - опорный элемент (pivot)
  - Зеленый - отсортированные элементы

### Вкладка "Теория/Код"
- Просмотр исходного кода файлов проекта
- Копирование кода в буфер обмена
- Файлы: CMakeLists.txt, main.cpp, sorting_visualizer.h, gpu_algorithms.cu, cpu_algorithms.cpp

### Вкладка "Отчеты"
- Текстовые отчеты о производительности
- Сравнение CPU vs GPU
- Описание метрик

- Вторичный фон: #27272a

## Отличия от оригинала

1. **Технология**: Qt 6 Widgets вместо React/Web
2. **Графики**: Qt Charts вместо Recharts
3. **Анимация**: QTimer вместо requestAnimationFrame
4. **Симуляция**: Эмуляция времени выполнения вместо реальных замеров
5. **Нет CUDA**: Для реальной работы GPU алгоритмов требуется подключение CUDA

## Расширение

Для добавления реальных измерений производительности:
1. Интегрируйте cpu_algorithms.cpp с реальными реализациями
2. Добавьте gpu_algorithms.cu для CUDA ядер
3. Замените BenchmarkSimulator на реальный BenchmarkRunner
