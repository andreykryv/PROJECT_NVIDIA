# SortBench: CUDA vs CPU

**SortBench: CUDA vs CPU** — это современный полнофункциональный интерактивный стенд для детального сравнения и анализа производительности алгоритмов сортировки, запущенных на процессоре (x86 CPU) и видеокарте (NVIDIA CUDA GPU). Позволяет визуализировать работу алгоритмов в режиме реального времени («по шагам» или плавно), проводить симуляцию аппаратных сред с физически точным моделированием латентности шины PCIe, температурного троттлинга и передаваемых байт данных, а также строить интерактивные аналитические графики.

---

## 🚀 Основные возможности

1. **Интерактивный визуализатор сортировок**:
   * Наглядная двухмерная визуализация выполнения алгоритмов.
   * Контроль над процессом: воспроизведение (`Play`), пауза (`Pause`), покадровый шаг вперед (`SkipForward`) и мгновенный сброс.
   * Настройка задержки выполнения (скорости шагов).

2. **Эмуляция аппаратного оборудования**:
   * Определение характеристик шины данных PCIe (Gen3, Gen4, Gen5 x16) с учётом времени передачи данных из оперативной памяти в видеопамять (Host to Device) и обратно (Device to Host).
   * Имитация температурного троттлинга GPU (Thermal Throttling) при нагреве ядер до 85°C, увеличивающего задержки вычислений.
   * Выбор разрядности данных: 32-битный `float` или 64-битный `double` (физический объём RAM увеличивается в 2 раза, влияя на пропускную способность шины).

3. **Стенд Бенчмарков (Аналитика)**:
   * Сравнение различных распределений исходных массивов: Случайное (Uniform Paint), Почти отсортированное (Nearly Sorted), Обратное (Reversed) и Сгруппированное (Few Unique).
   * Построение графиков времени вычисления, медианы, дисперсии и распределения задержек с помощью `Recharts`.
   * Экспорт результатов замеров в структурированный `JSON`, красивую `Markdown` таблицу или подробную `CSV` таблицу для дальнейшего анализа.

## 🛠️ Стек технологий

Проект разработан по гибридной архитектуре (Full-Stack SPA + Backend Server):

* **Frontend (Интерфейс)**:
  * **React 19** и **TypeScript** — современная компонентная архитектура с сильной типизацией.
  * **Vite 6** — быстрая сборка приложения.
  * **Tailwind CSS 4** — современный служебный CSS-фреймворк для стилизации интерфейса.
  * **Motion** (`motion/react`) — плавные анимации переходов, графических элементов и всплывающих уведомлений (Toast).
  * **Recharts** — интерактивные адаптивные графики для визуализации бенчмарков.
  * **Lucide React** — векторные иконки.

* **Backend (Сервер API)**:
  * **Express** (Node.js) — легковесный сервер для симуляции физических задержек выполнения алгоритмов сортировки на реальном CPU/GPU с учётом аппаратных параметров.
  * **tsx** и **esbuild** — высокопроизводительныеинструменты для выполнения и компиляции TypeScript кода сервера.

## ⚙️ Инструкция по запуску

### Предварительные требования

Для запуска и сборки проекта локально вам понадобятся:
* **Node.js** (версии 18 и выше)
* Менеджер пакетов **npm** (поставляется вместе с Node.js)

### 1. Установка зависимостей

Перейдите в корневую директорию проекта и установите все необходимые npm-пакеты:

```bash
npm install
```

### 2. Запуск в режиме разработки (Development)

Чтобы запустить клиент и сервер в dev-режиме с автоматической перезагрузкой при изменениях кода, введите команду:

```bash
npm run dev
```

После запуска сервер будет доступен по адресу: **http://localhost:3000**

### 3. Сборка для публикации (Production Build)

Для компиляции всего веб-приложения (сборка оптимизированного веб-интерфейса во временную папку `dist` и бандлинг Express сервера с помощью `esbuild` в единый файл `dist/server.cjs`):

```bash
npm run build
```

### 4. Запуск собранного приложения (Production Run)

После успешной команды сборки (`npm run build`), вы можете запустить готовое к работе полнофункциональное продакшн-приложение:

```bash
npm run start
```

Приложение будет работать на порту `3000`.

## 🧾 Доступные скрипты в `package.json`

Вы можете выполнять в терминале следующие скрипты:

* `npm run dev` — запуск приложения в режиме разработки (`tsx server.ts`).
* `npm run build` — сборка статики Vite и серверов в `dist/`.
* `npm run start` — запуск скомпилированного продакшн-сервера из `dist/server.cjs`.
* `npm run lint` — проверка TypeScript файлов на корректность типов (`tsc --noEmit`).
* `npm run clean` — очистка скомпилированных файлов и сборки.
